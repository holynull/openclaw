# 🔐 API Key 泄露应对指南

## ⚠️ 风险等级评估

### 如果 API Key + Secret 泄露：

| 风险项 | 风险等级 | 说明 |
|--------|---------|------|
| **绕过认证** | 🔴 高 | 攻击者可以通过认证 |
| **盗取资金** | 🟡 中 | 受金额限制和地址限制保护 |
| **业务中断** | 🟡 中 | 可能消耗配额导致正常业务受阻 |
| **数据泄露** | 🟢 低 | 只能获取签名服务数据，无法访问 HSM |

**总体风险**: 🟡 **中等（可控）**

---

## 🛡️ 多层防御体系

即使 API Key 泄露，攻击者仍需突破：

```
第1层：IP 白名单 ✅ (如果启用)
   ↓ 绕过
第2层：请求频率限制 ✅
   ↓ 绕过
第3层：异常检测 ✅ (新增)
   ↓ 绕过
第4层：地址黑白名单 ✅
   ↓ 绕过
第5层：金额限制 ✅
   ↓ 绕过
第6层：每日限额 ✅
   ↓ 绕过
第7层：审计日志 + 告警 ✅
```

**攻击者需要突破所有 7 层才能造成严重损失！**

---

## 🚨 新增的防护措施

### 1. **异常检测系统**

自动检测并封禁可疑行为：

- ✅ **请求频率异常**：1分钟内 >50 次请求 → +50 可疑分
- ✅ **多次失败**：短时间多次认证失败 → +10 可疑分
- ✅ **异常时间访问**：凌晨 2-6 点访问 → +3 可疑分
- ✅ **IP 地址变化**：同一用户 IP 频繁变化 → +10 可疑分
- ✅ **黑名单地址尝试**：尝试向黑名单转账 → +50 可疑分
- ✅ **达到 100 分**：自动封禁 30 分钟

### 2. **实时 Telegram 告警**

任何可疑行为立即通知：

```
🚨 User Blocked - Suspicious Activity

User: api_key_12345
Suspicious Score: 115
IP: 203.0.113.42
Block Duration: 30 minutes

Action Required: Review user activity
```

### 3. **API Key 轮换机制**

定期更换密钥，限制泄露影响：

```typescript
// 轮换 API Key（旧的 7 天后失效）
const newKey = rotateApiKey(oldApiKey, oldSecret, 7);
```

## 📋 发现泄露后的应急响应

### 立即行动（5 分钟内）

```bash
# 1. 立即撤销泄露的 API Key
curl -X POST http://localhost:3000/admin/revoke-key \
  -d '{"keyId": "leaked-key-id", "reason": "Compromised"}'

# 2. 检查审计日志
tail -100 logs/audit.log | grep "leaked-key-id"

# 3. 封禁可疑 IP
curl -X POST http://localhost:3000/admin/block-ip \
  -d '{"ip": "203.0.113.42"}'
```

### 短期行动（1 小时内）

1. **审查最近的交易**
2. **检查是否有资金损失**
3. **通知所有用户轮换密钥**

### 长期改进（1 周内）

1. **启用 IP 白名单**
2. **配置 Telegram 告警**
3. **降低金额限制**
4. **强制密钥轮换**

---

## 🎯 最佳实践

### 防止泄露

1. **永远不要在代码中硬编码**
2. **使用密钥管理服务**（Vault/AWS Secrets）
3. **限制密钥权限**
4. **定期轮换**（每月一次）

---

## 🎓 总结

### API Key 泄露 ≠ 完全失守

通过多层防护：
- ✅ 攻击者无法一次盗取大额资金
- ✅ 异常行为会被快速检测和封禁
- ✅ 所有操作都有审计日志可追溯
- ✅ 实时告警让你可以快速响应

**现在的系统：即使 API Key 泄露，也能将损失控制在可接受范围内！** 🛡️
